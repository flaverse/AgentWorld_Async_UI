## 二、叙事不合逻辑

### 1. 特莉丝说「好久不见」 (119s)
完整序列：14s 到吧台→18s 和杰洛特说话→28s 坐杰洛特旁边→57s 和杰洛特聊天→106s 讲故→**119s「杰洛特，好久不见」**。
她一直跟杰洛特在一起，没有离开过。这句话不合上下文。

### 2. 特莉丝的酒变了三次 (14s/28s/40s)
14s 要麦酒 → 28s 变成「放下蜂蜜酒」→ 40s 又变成葡萄酒。没有喝酒的动作就能自动变成不同的酒。

### 3. 丹德里恩「边弹唱边听故事」 (78s/85s)
78s 站起身弹唱叙事曲 → 85s **又**去酒吧听卓尔坦和维瑟米尔讲故事。自己在唱歌不可能同时听别人讲故事。

### 4. 丹德里恩反复「准备表演」 (37s/78s/110s)
37s 已弹唱《白果园的春天》→ 78s 「开始弹唱叙事曲」→ 110s 「准备表演」。已经唱了三次不同的歌，但还在准备。

### 5. 卓尔坦催唱但丹德里恩早已在唱 (55s-96s)
55s/67s/74s/78s/90s/96s 卓尔坦6次示意丹德里恩唱新曲。但丹德里恩 37s 和 78s 已经分别在弹唱了。卓尔坦的听觉层应该听到了却没有反应。

### 6. 杰洛特问狮鹫6次无人回答 (57s-143s)
57s→63s→122s→128s→131s→143s，6次问维瑟米尔狮鹫的事。维瑟米尔127s「准备讲」但从未真正讲出来。

### 7. 叶奈法煮茶仅用5秒 (143s/148s)
143s 提议煮茶 → 148s 「茶已煮好」。5秒现实时间对应约 5×60 = 300 sim-minutes。物理上可能，但时序读起来太快。

### 8. 维瑟米尔喊「听熊故事」但故事从未开始 (75s-99s)
75s 说「先听卓尔坦的熊故事」→ 81s 重复 → 99s 等卓尔坦讲。但110s 开始变成了故事接龙游戏，熊故事被遗忘了。

## 三、根本原因

### A. write-pending lock 只跳一帧
交互后 agent 跳过一帧（0.3s），但在 3-6s 后恢复，而场景没变。KL 相同触发 → LLM 给出相似输出。mute filter 只比较上一个输出，间隔一帧的非相邻重复全通过。

### B. 观察状态机没有「等到了回复才停止」
`expects_reply` 检出对方发新 speech 就标记 replied。但杰洛特问狮鹫 → 维瑟米尔说别的 → 检测到 speech 就算 replied。杰洛特不知道维瑟米尔没回答自己的问题，继续问。

### C. LLM 看不到历史对话内容
Agent 的 context 有 `memory_text` 但只是最近行动的摘要。LLM 不知道自己4秒前说过什么话、问过什么问题。所以它会重复「问狮鹫」——从它的视角，每一次都是第一次问。

### D. sensory 通道不干预 reasoning
卓尔坦听到丹德里恩弹唱 → auditory 层更新 → KL 触发。但 KL 文本是「auditory: 丹德里恩 变化」，不是「丹德里恩在唱歌」。Agent 知道"有变化"但不知道"是什么变化"。它只能重新调用 LLM 再决定一次。

## 四、总结

| 指标 | 数值 |
|------|------|
| 相邻完全重复 | 2次 |
| 语义近似重复（同一主题连续≥3次） | 9个 agent-topic 对 |
| 叙事逻辑矛盾 | 8处 |
| 主要根因 | write-pending锁太短 + mute只看上一个 + LLM无对话历史 |

修复优先级：**给 LLM context 注入最近3条对话摘要 > 延长 mute 窗口到 N=3 > 观察状态机区分「对方说话了」和「对方回答了我的问题」**
