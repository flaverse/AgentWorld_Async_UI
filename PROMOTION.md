## 二、小红书正文

> **标题**: 斯坦福论文要 3 次 LLM，我的只要 1 次。Agent 世界引擎开源。
>
> 🧠 做了一件事：把斯坦福 2023 年那篇《Generative Agents》的核心循环砍掉了 70% 的 Token 消耗。
>
> **核心想法很简单——能不调用 LLM 就不调用。**
>
> ━━━━━━━━━━━━━━━━
> **它和斯坦福那篇有什么不同？**
>
> 斯坦福的 Agent 按固定时间间隔反思 → 规划 → 行动。
> 我的 Agent **只有世界发生变化时才思考**。
>
> 原理是用 KL 散度持续比较"Agent 记忆中的世界"和"真实世界"——
> 四通道并行监听（听觉/视觉/状态/时间），任意一个变了才触发 LLM。
>
> 结果：
> ✅ NPC 互相对话：1 次 LLM（斯坦福要 3+）
> ✅ 发呆观察：0 次 LLM
> ✅ NPC 使用物品：2 次 LLM
>
> ━━━━━━━━━━━━━━━━
> **为什么是"零代码换世界"**
>
> 所有规则、Prompt、阈值、货币、属性全部在 YAML 里。
> 换一个 `world.yaml` = 换一个世界。
> Python 源码一个字不用改。
>
> 猎魔人酒馆 → 赛博朋克夜之城 → 霍格沃茨大厅
> 改 YAML 即可。
>
> ━━━━━━━━━━━━━━━━
> **架构上做了什么**
>
> 🔹 分层架构：视觉/听觉/交互层各自独立，Agent 通过轮询感知世界，不需要事件总线
> 🔹 相位流水线：SENSE → OBSERVE → KL GATE → DECIDE → ACT，每步可独立跳过
> 🔹 Agent 状态归位：11 个 agent 专属字段从通用 Entity 迁移到 AgentLayer，桶子不再有"对话记忆"
> 🔹 全模态记忆：LLM 输出的对话、表情、内心独白、属性变化以 JSON 完整存储
>
> ━━━━━━━━━━━━━━━━
> **数据说话（180s 运行）**
>
> · 8 个 Agent ·128 次行动 ·41 次 NPC 互相对话 ·93% 对话响应率
> · 重复率仅 3.3%（带去重滤波器）
> · 4 种输出模态全激活（Story/Dialogue/Visual/Internal）
>
> ━━━━━━━━━━━━━━━━
> **GitHub 开源** → 搜 AgentWorld Async
> **配置**: DeepSeek / OpenAI / MiniMax 任意
> **一行跑**: `python main.py`
